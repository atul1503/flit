# Flit project guide for Claude

A Flutter-inspired UI toolkit for Nim. Single codebase targets macOS, Linux, Windows, iOS, Android, web, and embedded Linux. See `README.md` for the user-facing pitch and `docs/ARCHITECTURE.md` for the design.

## Repo layout

- `src/flit.nim` is the top-level umbrella import (`import flit`).
- `src/flit/foundation/` holds the framework primitives. Touch carefully.
- `src/flit/rendering/` holds RenderObjects.
- `src/flit/widgets/basic.nim` holds the layout widgets (Container, Row, Column, etc.).
- `src/flit/material/` and `src/flit/cupertino/` are design system wrappers.
- `src/flit/platform/{desktop,web,mobile,embedded}/runner.nim` are the per-target event loops.
- `cli/src/flit_cli.nim` is the `flit` command.

## Adding a new widget

1. Decide: is it stateless, stateful, or render-object?
2. For render widgets: define a `RenderXxx` in `rendering/`, then a `Xxx` widget in `widgets/basic.nim` (or in a design module). Override `widgetTypeName`, `createElement`, `createRenderObject`, and `updateRenderObject`.
3. If the widget has children, teach `childrenOf` in `foundation/runtime.nim` how to extract them.
4. If the child needs special parent data (like Flexible's `flex`), extend `attachChildRenders`.

## Versioning

Semver. Bump before publishing user-facing changes. `flitVersion` lives in `cli/src/flit_cli.nim`; `version` lives in `flit.nimble`. Keep them in lockstep. Version history below.

- 0.9.7: Defensive correctness pass. Hunted for bug classes beyond iteration safety. Fixed: (1) **UTF-8 byte indexing in TextField**: cursor moved 1 byte instead of 1 codepoint, so backspace on `é` (2 bytes) or 👍 (4 bytes) corrupted the string. Added `utf8Back` / `utf8Forward` helpers that skip continuation bytes (0x80..0xBF). Fixed `backspace`, `forwardDelete`, `moveLeft`, `moveRight`. (2) **Div-by-zero in `newAnimationController`**: if `durationSec <= 0`, the frame ticker divided elapsed time by zero producing NaN, which propagated through `clamp` and broke listeners. Now clamped to 1ms minimum. (3) **Div-by-zero in embedded runner**: `1000 div frameRateHz` crashed if user passed 0. Now `max(frameRateHz, 1)`. (4) **Wrong guard in `RenderImage.performLayout`**: code checked `imgW > 0` then divided by `imgH`. Fixed to check the actual divisor. Also added zero-dim early-out in paint to avoid `sw/sh` / `dw/dh` div-by-zero. (5) **GPU memory leaks**: `SdlSubCanvas.texture`, `GpuSubCanvas.texture` never destroyed on unmount; `GpuCanvas.shapeCache` grew unbounded. Added `disposeSubCanvas` for both and `trimShapeCache(maxEntries)` for the latter. 20 new regression tests across `test_text_field.nim` (UTF-8) and `test_edge_inputs.nim` (degenerate inputs). 237 total tests, all green (up from 217).
- 0.9.6: Iteration-safety audit. The 0.9.5 fix covered `ValueNotifier` listeners but missed five other observer-style loops that could crash with `AssertionDefect: the length of the seq changed while iterating over it` when callbacks fired during notification mutated the collection. Fixed: (1) **All four platform runners** (desktop, web, mobile, embedded) iterated `binding.dirtyRoots` directly. `rebuildElement` can add to dirtyRoots via `InheritedWidget` notifications that propagate to descendants, or via setState callbacks fired by listeners during the rebuild. Now snapshot + clear before iterating; new dirty roots process on the next frame. (2) `AnimationController.value=` / `setValue` / `setStatus` iterated listeners directly. Snapshot. (3) `TextEditingController.value=` / `restore` iterated listeners directly. Snapshot. (4) `notifyInheritedDependents` iterated `e.dependents` directly. Snapshot. (5) Public `visit` / `visitDeep` helpers iterated `e.children`. Defensive snapshot for user callbacks. New `tests/test_iteration_safety.nim` with 6 regression tests exercising each pattern. 217 tests, all green. Cold benchmark went from 0.85ms to 0.93ms (the snapshot copies cost ~70µs/frame; still 82x faster than Flutter cold). Worth the small cost for crash-free operation under realistic listener patterns.
- 0.9.5: Bugfix. Clicking "Set to Bob" (or any state-mutating button in the showcase State tab) crashed with `AssertionDefect: the length of the seq changed while iterating over it`. Root cause: `ValueNotifier.value=` and `.notify` iterate `n.listeners` directly while invoking each listener. If a listener triggers a setState that rebuilds the widget tree, new ListenableBuilders mount and call `addListener`, growing the seq mid-iteration. Fix: snapshot the listeners seq before iterating. New listeners registered during a notification take effect on the next notification, not the current one. Matches Flutter's `ChangeNotifier` semantics. All 211 tests still green.
- 0.9.4: Bugfix. Clicking the Style tab in the showcase example crashed with `IndexDefect: index 786434 not in 0 .. 786431`. Root cause: `RenderClipRRect.paint` calls `ctx.canvas.clipRect(rectFromOffsetSize(offset, r.size))` which can hand Pixie a clip rect that extends past the canvas buffer when the widget is laid out near or past the edge. Pixie's NEON-accelerated path fill then walks off the end of the pixel buffer. Fix: clamp the clip rect to canvas bounds in `SdlCanvas.clipRect` and `SdlSubCanvas.clipRect`. Safe because nothing outside the canvas is visible anyway. Reproduced headlessly first via a small test that mounted Showcase and switched to tabStyle, then verified the live SDL run no longer crashes. All 211 tests still green.
- 0.9.3: Warm-path performance pass. Two caches added that together drop both cold and warm benchmark times by ~36x. (1) Text measurement cache in `rendering/text.nim`: `wrapMeasureWithCache(fn)` returns a memoizing version of any `measureText` proc, keyed by `(text, fontFamily, fontSize, fontWeight)`. Layout dropped from 14.4 ms to 0.17 ms for the 500-card workload. (2) Text rasterization cache in `platform/embedded/runner.nim`: rendered text bitmaps cached as Pixie images, keyed by `(text, fontSize, color)`. Subsequent draws blit the cached image instead of re-rasterizing. Paint dropped from 14.8 ms to 0.60 ms. New benchmark numbers (apples-to-apples vs Flutter 3.24.4 on Apple Silicon): **flit cold 0.85 ms vs Flutter cold 76.4 ms (90x faster); flit warm 0.77 ms vs Flutter warm 7.5 ms (10x faster, and flit includes rasterization while Flutter test mode doesn't).** 211 tests still green. benchmarks/README.md updated with historical progression table.
- 0.9.2: First real benchmark vs Flutter. New `benchmarks/` directory with apples-to-apples 500-card workload in both flit (Pixie embedded canvas) and Flutter (test renderer). 200 iterations, 30 warmup. **flit cold** (fresh widget tree per iter, full rebuild): **31 ms** vs Flutter's 76 ms — flit wins ~2.4x on the path every setState triggers. **flit warm** (existing tree, full subtree invalidation via recursive `invalidateSubtree`): 29 ms vs Flutter's 7.5 ms, but Flutter test mode doesn't rasterize pixels while flit does, so the warm comparison is asymmetric. With rasterization equalized the warm path is roughly comparable; flit's layout bottleneck is Pixie text measurement (every text widget calls `typeset` during layout). `benchmarks/README.md` documents methodology, the rasterization asymmetry, and per-phase breakdown in detail. The earlier "Flutter is faster" claim was wrong; the corrected story is: flit beats Flutter on the rebuild path and ties on steady-state.
