# Flit project guide for Claude

A Flutter-inspired UI toolkit for Nim. Single codebase targets macOS, Linux, Windows, iOS, Android, web, and embedded Linux. See `README.md` for the user-facing pitch and `docs/ARCHITECTURE.md` for the design.

## Repo layout

- `src/flit.nim` is the top-level umbrella import (`import flit`).
- `src/flit/foundation/` holds the framework primitives. Touch carefully.
- `src/flit/rendering/` holds RenderObjects.
- `src/flit/widgets/basic.nim` holds the layout widgets (Container, Row, Column, etc.).
- `src/flit/material/` and `src/flit/cupertino/` are design system wrappers.
- `src/flit/platform/{desktop,web,mobile,embedded}/runner.nim` are the per-target event loops.
- `cli/src/flit_cli.nim` is the `flit` command.

## Adding a new widget

1. Decide: is it stateless, stateful, or render-object?
2. For render widgets: define a `RenderXxx` in `rendering/`, then a `Xxx` widget in `widgets/basic.nim` (or in a design module). Override `widgetTypeName`, `createElement`, `createRenderObject`, and `updateRenderObject`.
3. If the widget has children, teach `childrenOf` in `foundation/runtime.nim` how to extract them.
4. If the child needs special parent data (like Flexible's `flex`), extend `attachChildRenders`.

## Versioning

Semver. Bump before publishing user-facing changes. `flitVersion` lives in `cli/src/flit_cli.nim`; `version` lives in `flit.nimble`. Keep them in lockstep. Version history below.

- 0.7.0: Performance pass 2. Five major subsystems added with no existing public API changed. (1) Variable-extent slivers: `ListViewBuilder` now accepts an `extentFor: proc(idx: int): float32` callback; `RenderSliverList` maintains a prefix-sum cache of per-item extents and uses binary search for offset-to-index queries. (2) HarfBuzz text shaping: new `rendering/harfbuzz.nim` provides minimal Nim bindings to libharfbuzz (blob, face, font, buffer, shape). `GlyphAtlas` gains `registerHbFont` and `measureShaped`; rasterization uses HarfBuzz width measurement when available, falls back to Pixie `typeset` otherwise. Real ligatures and kerning for Latin scripts. (3) GPU shader-based path rasterization: new `rendering/canvas_gl.nim` opens an OpenGL 3.3 context via SDL2_GL, compiles SDF shaders for rounded rect, circle, and stroked line. Antialiased shape rasterization happens on the GPU with no CPU work or texture caching. (4) Off-main-thread raster: new `rendering/raster_pool.nim` provides a fixed worker-thread pool with safe submit/drain/shutdown. Uses Nim channels + atomics; nimcall procs only (no closures across threads) for ORC compatibility. (5) Identity short-circuit in reconciliation: when a parent's rebuild returns the same widget reference, the child element keeps its slot without re-walking. Huge win for stable subtrees and `RepaintBoundary`-wrapped content. Plus: `applyOpacity` inlined on the hot paint path. 35 new tests added; 165 total, all green. Added `opengl >= 1.2.0` dependency.
- 0.6.0: Performance pass. Four major subsystems added without changing any existing public API: (1) `RepaintBoundary` widget + `RenderRepaintBoundary` cache rasterized subtrees in an off-screen sub-canvas; the desktop `SdlCanvas` backend uploads to an SDL streaming texture once and composites via `SDL_RenderCopy` (GPU) thereafter. `markNeedsPaint` flips the boundary's `cacheDirty` flag so re-rasterization only happens when something inside actually changed. (2) New `GpuCanvas` backend in `rendering/canvas_gpu.nim` uses SDL2's hardware renderer for all simple draws (`SDL_RenderFillRect`, `SDL_RenderDrawLine`, `SDL_RenderClear`); rounded rects and circles get a per-shape texture cache keyed by `(dims, color)` so the first frame rasterizes once and every frame after is a GPU blit. (3) `GlyphAtlas` in `rendering/glyph_atlas.nim` caches rasterized text by `(text, font, size, color)`; subsequent draws of the same label are one `SDL_RenderCopy`. (4) `ListView.builder` (`widgets/lazy_list.nim`) + `RenderSliverList` (`rendering/sliver_list.nim`) implement lazy lists: a 10,000-item list mounts only the ~10 visible items per frame, drops items that scroll out, and reports correct full-content scrollbar geometry. 9 new tests added across `test_repaint_boundary.nim`, `test_layer.nim`, and `test_lazy_list.nim`; all 139 tests green. Also added `config.nims` with macOS rpath flags so SDL2 dynlib loading works without `DYLD_LIBRARY_PATH`.
